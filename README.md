# ICU Trajectory Deterioration Detection System

> Early detection of physiological deterioration in ICU patients using a hybrid ensemble of machine learning, deep learning sequence models, and rule-based trajectory logic — trained on 12,564 real ICU patients from the PhysioNet Challenge 2019 dataset.

## What It Does

Monitors ICU patient vital signs over time and flags physiological deterioration up to 12 hours before it becomes critical. Rather than predicting a single condition like sepsis, the system detects any dangerous deviation across:

- Heart Rate below 40 or above 120 bpm
- Systolic BP below 90 mmHg
- SpO2 below 94%
- Respiratory Rate above 24 or below 8 per min
- Temperature above 38.5C or below 35.5C

This produces a 26% positive rate vs 5% for sepsis-only labels, enabling the GRU to learn meaningful trajectory patterns.

                |

                |

                |

## Dataset

**PhysioNet Challenge 2019** - Early Prediction of Sepsis from Clinical Data

- 12,564 ICU patients | 489,430 hourly records
- Features: HR, SBP, DBP, SpO2, RR, Temperature + lab markers
- Patient-level train/val/test split to prevent data leakage

## Key Design Decisions

**Deterioration labels over sepsis labels**
Vital-sign clinical thresholds increase positive rate from 5% to 26%, giving deep learning models enough signal to learn real trajectory patterns.

**Patient-level splitting**
Split by patient ID, not by window. Prevents the model from memorising patient-specific patterns and inflating metrics.

**GRU over LSTM**
GRU converges faster and achieves higher AUC on short 12-hour sequences. Fewer parameters (157k vs 208k) with better generalisation on this dataset.

**Hybrid ensemble**
Rule-based trajectory logic provides clinical interpretability. ML models handle tabular features. GRU handles raw temporal sequences.

Built during a research internship at the **Indian Institute of Science (IISc), Bengaluru**. Proof-of-concept research prototype demonstrating clinical viability pending prospective validation and ethics approval.

The GRU model achieves AUC-ROC of 0.955 on held-out test data, competitive with hospital-deployed clinical decision support systems, trained entirely on publicly available data using a single GPU.
